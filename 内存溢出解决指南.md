# 内存溢出问题解决指南

## 问题症状
- Next.js 编译时内存溢出
- 开发服务器启动失败
- 构建过程卡死

## 解决方案

### 方案一：使用优化启动脚本（推荐）

直接双击运行 `启动网站.bat` 文件，该脚本会自动：
- 设置 Node.js 内存限制为 4GB
- 启动开发服务器

### 方案二：手动启动

在项目目录下运行：
```bash
# 设置内存限制后启动
set NODE_OPTIONS=--max-old-space-size=4096
npm run dev
```

### 方案三：清理缓存后重启

1. 双击运行 `清理缓存.bat`
2. 删除以下目录（如果存在）：
   - `.next`
   - `node_modules/.cache`
3. 重新启动：`启动网站.bat`

### 方案四：使用 npm 脚本（需要安装 cross-env）

```bash
# 首次安装 cross-env
npm install --save-dev cross-env

# 使用低内存启动脚本
npm run dev:low-memory
```

## 如果问题仍然存在

1. **增加内存限制**（如果有足够物理内存）：
   ```batch
   set NODE_OPTIONS=--max-old-space-size=6144
   ```

2. **检查占用内存的进程**：
   ```batch
   tasklist /FI "IMAGENAME eq node.exe"
   ```

3. **升级到最新版本**：
   ```bash
   npm update
   ```

4. **使用 Turbopack 替代 Webpack**：
   ```batch
   npm run dev --turbo
   ```
