## 可能的原因

### 1. Next.js 编译中（最可能）
**症状：** 首次启动后访问主页显示空白或错误
**原因：** Next.js 正在编译页面，需要时间
**解决方法：**
- 等待 1-3 分钟让编译完成
- 刷新浏览器页面
- 查看终端是否有编译完成的消息

### 2. 组件或依赖错误
**症状：** 编译失败，浏览器控制台有错误信息
**原因：** 缺少依赖包或组件引用错误
**检查步骤：**
```bash
# 检查依赖是否完整
npm install

# 检查是否有编译错误
npm run build
```

### 3. 数据库连接问题
**症状：** 页面加载但数据不显示
**原因：** API 路由无法连接到数据库
**解决方法：**
- 检查 Prisma 配置
- 确保数据库服务正在运行

## 常见错误及解决方法

### 错误 1: Module not found
```
解决: npm install <missing-package>
```

### 错误 2: Cannot find module '@/components/...'
```
解决: 检查 tsconfig.json 的 paths 配置
```

### 错误 3: Port 3000 already in use
```
解决: 
taskkill /F /IM node.exe
或更改端口: npm run dev -p 3001
```

### 错误 4: ENOMEM 内存不足
```
解决: 使用内存限制启动
set NODE_OPTIONS=--max-old-space-size=4096
npm run dev
```
