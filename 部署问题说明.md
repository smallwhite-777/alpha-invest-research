## 当前问题：open1nvest.com 无法访问

### 问题现象

访问 https://open1nvest.com 返回：
```
Error code: 525
SSL Handshake Failed
```

### DNS 配置

**Cloudflare DNS记录：**
| 类型 | 名称 | 内容 | 代理状态 |
|------|------|------|----------|
| A | @ | 76.76.21.21 | 已尝试 DNS only / Proxied |
| CNAME | alpha-backend | alpha-invest-backend.cfargotunnel.com | Proxied |

**DNS解析结果：**
```
$ nslookup open1nvest.com
Address: 198.18.0.125  # 这个IP是Cloudflare的代理IP，不是Vercel的IP
```

### 已尝试的解决方案

1. ✅ 在Vercel添加了 open1nvest.com 域名，状态显示 **Valid** (绿色)
2. ✅ Cloudflare SSL模式设置为 **Full**
3. ❌ 尝试关闭Cloudflare代理（DNS only），仍未生效
4. ❌ DNS解析仍然指向 198.18.0.125 而不是 Vercel IP

### 可能的原因

1. **DNS传播延迟** - 修改DNS后需要等待
2. **Cloudflare代理冲突** - Cloudflare代理可能与Vercel的SSL冲突
3. **DNS记录缓存** - 本地DNS缓存了旧记录

### 期望结果

- `open1nvest.com` → Vercel网站 (Next.js前端)
- `alpha-backend.open1nvest.com` → Python后端 (Tunnel)

## 建议的排查步骤

1. **检查DNS解析是否正确指向Vercel**
   - 应该解析到 Vercel 的IP (76.76.21.21 或其他Vercel IP)
   - 当前解析到 198.18.0.125 (Cloudflare IP)

2. **尝试完全关闭Cloudflare代理**
   - DNS记录设置为 "DNS only" (灰色云朵)
   - 等待DNS传播 (可能需要数小时)

3. **或者使用Cloudflare代理 + 正确SSL配置**
   - SSL模式设为 "Full (strict)"
   - 确保Vercel域名SSL证书有效

4. **清除本地DNS缓存**
   ```powershell
   ipconfig /flushdns
   ```
